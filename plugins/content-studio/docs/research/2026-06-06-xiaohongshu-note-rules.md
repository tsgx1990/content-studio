# 小红书笔记格式规范（for the xhs-post gate）

- **日期：** 2026-06-06
- **主题：** 小红书（XHS）图文笔记的可机械校验规范（标题/正文字数、话题标签数、极限词限流），
  用于 `scripts/check-xhsnote.mjs`（xhs-post 垂直线的 deterministic gate）。
- **数据源：** SearXNG（language=zh），多篇中文来源交叉印证。

## 问题

哪些小红书发布规则是**客观、可复现校验**的（不是审美判断），从而能进 gate 强制执行，而不是靠
模型自觉？

## 关键发现（多源一致）

- **标题：** 普通笔记标题 **最多 20 个字符**（含汉字/字母/数字），超出会被**截断**；搜索权重集
  中在**前 10~18 字**（“标题前 10 个字权重占比超 60%”）。
- **正文：** **最多 1000 字**（含空格、标点、表情）；2025 原创新规：**短笔记 ≥100 字**、长笔记
  ≥600 字；**600–800 字完读率最高**。
- **话题标签：** 平台**硬上限 30 个**；但实操强调“少而精”，**堆砌话题/关键词会触发限流**。
- **极限词限流（重要）：** 标题/正文出现**绝对化用语**（“最”“最好”“第一”“100%”“国家级”
  “最低价”“销量第一”等，《广告法》禁用词 + 平台违禁词）会**触发限流**，是常见的“小眼睛不过百”
  原因。这是可机械校验的。
- **导流限流/封号（最严，2024-11 起持续收紧 — 补充于 2026-06-06）：** 在笔记/评论/资料/私信中留
  下或**暗示**站外联系方式属于**违规导流**，处罚比限流更重（功能限制乃至封号）。官方《交易导流违规
  管理细则》（2025-03-12 生效）明确违规类型含“**传播个人/社群联系方式或其他平台账号/店铺信息**”。
  高置信、可机械校验的硬信号：**微信/微信号/weixin/wechat/加微/vx/v信/威信/薇信、QQ号/扣扣、公众号、
  二维码、手机号（11 位）**。软信号（“私信”“看主页”“简介”“联系我”）平台也会判，但**歧义高、正常
  笔记常用**——**刻意不进默认表**以免误伤（gate 求高精度，宁漏勿误）。
- 图片：最多 18 张，图文笔记建议 **3–9 张**（封面权重高）。

## 给 gate 的决策（defaults，均可在 note 的 `limits`/`banned_terms` 覆盖）

| 校验项 | 规则 | 依据 |
|--------|------|------|
| 标题字符数 | 5 ≤ len ≤ **20** | 平台硬上限 20；下限防空洞标题 |
| 正文字符数 | **100** ≤ len ≤ **1000** | 原创下限 100；硬上限 1000 |
| 话题标签数 | 1 ≤ n ≤ **10**（默认） | ≥1 才有曝光；默认 10 取“少而精”，平台硬上限 30 可覆盖 |
| 极限词 | **命中即失败** | 绝对化用语触发限流；blocklist 可覆盖 |
| 导流词 | **命中即失败** | 站外联系方式（微信/QQ/二维码/公众号/手机号）触发限流/封号；`diversion_terms` 可覆盖、`[]` 可关 |

字符计数用**码点数**（`Array.from(str).length`）近似平台“字符”口径（汉字记 1；emoji 可能记
为多个码点——近似，gate 求“大致合规”而非逐字节精确）。

极限词默认表用**多字词**（“最好/最佳/最低价/第一名/全网第一/销量第一/100%/国家级/顶级/绝对/
根治…”），**刻意不含单字“最”**——否则会误伤“最近/最后/最初”（与 readability gate 早期
“war”误伤“warm”同类陷阱）。

## 局限 / 诚实

- 平台规则**经常变**（字数、原创下限、违禁词每月都在调），defaults 是某一时点的快照，发布前应
  再核一次；故 note 支持 `limits`/`banned_terms` 覆盖。
- gate 是**合规底线**（不被截断、不堆砌、不踩限流词），**不**判断笔记“爆不爆”——选题/钩子/封面
  质量仍归 `content-review`。
- 极限词为子串近似匹配（中文无词边界），可能漏掉变体、或在极少数语境误伤；用多字词降低误伤。
